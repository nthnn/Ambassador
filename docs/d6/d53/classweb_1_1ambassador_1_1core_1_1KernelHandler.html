Detailed Description

The KernelHandler class is responsible for handling incoming HTTP requests and returning appropriate responses.

This class implements the HttpHandler interface and is part of the server's request handling pipeline. It interacts with the Router, which is used to process the request and determine the correct view/content to return. Additionally, it manages the database connection through the DatabaseManager and handles error responses using the provided ErrorController implementations.

The KernelHandler class is designed to handle various HTTP methods (e.g., GET, POST), parse request headers, cookies, and query parameters, and generate corresponding responses. If an error occurs during request processing, it will handle the error gracefully by using the appropriate error controller.

In the event of an exception or error, a default error handler is used if no specific error controller is provided for the given HTTP status code.

void web.ambassador.core.KernelHandler.handle ( HttpExchange  exchange) throws IOException
inline

Handles an incoming HTTP request by parsing the request details, processing the request using the router, and returning an appropriate response.

This method performs the following steps:

  1. Creates a new Request and Response object.
  2. Parses request details such as path, method, headers, cookies, query parameters, and body (for POST requests).
  3. Processes the request using the Router to determine the appropriate view or content.
  4. If the response status code is not 200, an error response is generated using the appropriate error controller.
  5. Writes the response to the HTTP exchange, including headers and response body.

In case of an exception during request processing, the method will respond with a generic 500 Internal Server Error, using the error controller if defined.

Parameters
exchangeThe HttpExchange object representing the incoming HTTP request and response.
Exceptions
IOExceptionIf an I/O error occurs while handling the request or response.
