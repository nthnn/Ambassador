19/**
20 * Core components and infrastructure for the Ambassador web server framework.
21 *
22 * The `web.ambassador.core` package provides the essential classes and mechanisms to build and manage
23 * a lightweight, high-performance HTTP server. This package serves as the backbone for the Ambassador
24 * web framework, facilitating server configuration, request routing, error handling, and lifecycle
25 * management.
26 *
27 * <h2>Overview</h2>
28 *
29 * This package defines the central Kernel class, which orchestrates the initialization and management
30 * of the HTTP server. It integrates components such as routing, database management, and custom error
31 * handling to deliver a comprehensive and extensible web server environment.
32 *
33 * The core responsibilities of this package include:
34 * <ul>
35 * <li>Bootstrapping and configuring the HTTP server.</li>
36 * <li>Managing request routing through dynamic controller registration.</li>
37 * <li>Handling errors and providing customizable error controllers.</li>
38 * <li>Integrating with the database layer through DatabaseManager.</li>
39 * <li>Ensuring graceful shutdown and resource management using the Cleaner API.</li>
40 * </ul>
41 *
42 * <h2>Key Classes</h2>
43 *
44 * <h3>Kernel</h3>
45 * <ul>
46 * <li>The main entry point for configuring and starting the HTTP server.</li>
47 * <li>Responsible for initializing thread pools, binding server ports, and managing the request lifecycle.</li>
48 * <li>Implements {@code java.lang.AutoCloseable} to ensure proper resource cleanup.</li>
49 * <li>Supports custom shutdown hooks, error controllers, and dynamic route registration.</li>
50 * </ul>
51 *
52 * <h3>Router</h3>
53 * <ul>
54 * <li>Facilitates the registration and mapping of controller classes to specific HTTP routes.</li>
55 * <li>Handles dispatching of requests to the appropriate controller methods.</li>
56 * <li>Central to the Ambassador framework's ability to handle RESTful APIs dynamically.</li>
57 * </ul>
58 *
59 * <h3>KernelConfig</h3>
60 * <ul>
61 * <li>Provides a convenient configuration object for initializing the Kernel with custom settings.</li>
62 * <li>Allows easy specification of port numbers, thread pools, database connections, and backlog sizes.</li>
63 * </ul>
64 *
65 * <h3>DatabaseManager</h3>
66 * <ul>
67 * <li>Handles database connectivity and connection pooling.</li>
68 * <li>Seamlessly integrates with Kernel to provide persistent storage support.</li>
69 * <li>Manages lifecycle operations, including connection establishment and graceful shutdown.</li>
70 * </ul>
71 *
72 * <h3>ErrorController</h3>
73 * <ul>
74 * <li>Represents a base class for handling error responses for different HTTP status codes.</li>
75 * <li>Developers can extend this class to create custom error pages or JSON error responses.</li>
76 * </ul>
77 *
78 * <h3>DefaultErrorController</h3>
79 * <ul>
80 * <li>A fallback error controller used when no specific error handler is registered.</li>
81 * <li>Provides basic HTTP error responses for common error scenarios.</li>
82 * </ul>
83 *
84 * <p><b>Usage</b></p>
85 * The following example demonstrates how to create and start a simple HTTP server using the Kernel:
86 *
87 * ```java
88 * Kernel kernel = new Kernel(8080);
89 * kernel.registerController(MyController.class);
90 * kernel.setupShutdownHook(
91 * () -> System.out.println("Server stopped successfully."),
92 * () -> System.err.println("Server shutdown failed.")
93 * );
94 * kernel.start();
95 * ```
96 *
97 * <h2>Shutdown and Resource Management</h2>
98 *
99 * The Kernel class ensures that resources are released correctly during shutdown by implementing the
100 * {@code java.lang.AutoCloseable} interface. This allows the Kernel to be used in try-with-resources
101 * blocks to guarantee safe shutdown even in the presence of exceptions.
102 *
103 * <p><b>Example:</b></p>
104 * ```java
105 * try (Kernel kernel = new Kernel(8080)) {
106 * kernel.start();
107 * }
108 * // Server shuts down automatically
109 * ```
110 *
111 * <h2>Error Handling</h2>
112 * Developers can register custom error handlers to manage specific HTTP status codes.
113 * This enables greater flexibility in defining server responses for various error conditions.
114 *
115 * <p><b>Example:</b></p>
116 * ```java
117 * kernel.setErrorController(404, new NotFoundErrorController());
118 * kernel.setErrorController(500, new InternalServerErrorController());
119 * ```
120 *
121 * <h2>Extensibility</h2>
122 *
123 * The package is designed to be extensible by allowing developers to create and register their own
124 * controllers, error handlers, and database configurations. This makes it suitable for building
125 * large-scale web applications with custom requirements.
126 *
127 * <h2>Concurrency and Scalability</h2>
128 *
129 * The Kernel utilizes a thread pool for concurrent request processing. By default, it scales with
130 * the number of available processors, ensuring efficient handling of multiple requests simultaneously.
131 * This behavior can be adjusted using the {@link web.ambassador.core.Kernel#setMaxThreads(int)} method.
132 *
133 * <p><b>Example:</b></p>
134 * ```java
135 * kernel.setMaxThreads(100);
136 * ```
137 *
138 * <h2>Notes</h2>
139 *
140 * <ul>
141 * <li>This package does not handle HTTPS directly. SSL/TLS support must be implemented separately.</li>
142 * <li>This package assumes a basic understanding of HTTP and RESTful API development.</li>
143 * </ul>
144 * <br/>
145 * This package is part of the larger Ambassador framework, which provides a set of tools for building web
146 * applications and handling HTTP communication. It offers a flexible way to handle the complexities of HTTP
147 * interactions in a modular and easy-to-understand manner.
